# AI 图片压缩工具

AI 图片压缩工具是一款本地运行的桌面图片压缩应用，基于 Electron、Vue 3、Pinia 和 Sharp 构建。图片读取、压缩、导出和历史记录都在本机完成，不会上传图片或压缩结果。

## 功能

- 支持单张、多张和文件夹导入，支持拖拽导入。
- 支持 JPG、JPEG、PNG、WebP、BMP、GIF 读取；GIF 会按首帧压缩输出。
- 支持质量压缩、输出格式选择、按比例缩放和最大宽高限制。
- 支持单张保存和批量导出，批量导出会自动避开同名文件冲突。
- 支持压缩历史记录、搜索筛选、JSON 导入导出和从历史记录重新加入工作区。
- 支持压缩过程取消、导入进度展示和大文件内存高水位提示。

## 环境要求

- Node.js 20 或更高版本。
- pnpm 9 或更高版本。
- macOS、Windows 或 Linux 桌面环境。

## 安装与启动

```bash
pnpm install
pnpm run dev
```

开发服务会启动 Electron 桌面窗口。首次安装会通过 `postinstall` 执行 Electron 原生依赖安装。

## 构建与打包

```bash
pnpm run build
pnpm run build:unpack
pnpm run build:mac
pnpm run build:win
```

- `pnpm run build`：构建主进程、预加载脚本和渲染进程。
- `pnpm run build:unpack`：生成未打包的可执行目录，适合做安装态 smoke。
- `pnpm run build:mac`：生成 macOS 安装产物。
- `pnpm run build:win`：生成 Windows 安装产物。

## 测试

```bash
pnpm test
```

当前测试覆盖文件名生成、图片工作区 store 的选择/排序/独立参数/导入进度，以及历史记录 store 的日期筛选、搜索和统计。

## 使用说明

1. 点击上传区域选择图片，或点击“选择文件夹”批量导入目录中的图片。
2. 在右侧参数面板选择压缩质量、输出格式和尺寸限制。
3. 点击“开始压缩”等待任务完成；批量任务会逐张处理。
4. 压缩完成后，可保存当前选中图片，也可点击“导出全部”选择输出文件夹。
5. 打开历史记录面板可查看过往压缩结果、搜索、筛选、删除、导入或导出历史。

## 导入与导出规则

- 导入阶段会过滤重复路径，并对每个文件读取元数据和大小。
- 单文件大小上限为 100MB。
- 批量导出默认使用 `{原文件名}_compressed.{输出格式}`。
- 批量导出遇到同名文件时会自动追加序号，避免覆盖已有文件。

## 数据与隐私

- 图片压缩完全本地执行，不上传任何图片、路径或历史数据。
- 压缩临时文件保存在系统临时目录下的 `ai-images-tools` 文件夹。
- 历史记录保存在 Electron `app.getPath('userData')` 对应目录内的 `history.json`。
- 退出应用时会尝试清理临时压缩文件；异常退出后可手动删除系统临时目录下的 `ai-images-tools`。

## 常见问题

### 为什么 GIF 输出不是动图？

当前压缩链路只读取 GIF 首帧，并默认输出为 WebP 或指定格式。动图压缩不在当前功能边界内。

### 点击取消后为什么还有错误提示？

取消会向当前 Sharp pipeline 发送中断信号。大图任务中断后会走错误事件返回“压缩已取消”或“已取消”，不会写入成功结果或历史记录。

### 大批量图片压缩时提示内存较高怎么办？

建议分批导入和压缩大图，或降低最大宽高后再压缩。应用会记录本轮压缩的 RSS 内存高水位，超过阈值时给出提示。
