## 三、IPC 通信通道图

```mermaid
sequenceDiagram
    participant R as Renderer (Vue 3)
    participant P as Preload<br/>(contextBridge)
    participant M as Main Process<br/>(Node.js)
    participant S as Services<br/>(Sharp / fs)

    Note over R,M: ─── Renderer → Main (invoke) ───

    R->>P: compress:start / compress:batch
    P->>M: ipcRenderer.invoke
    M->>S: 调用压缩服务
    S-->>M: 返回结果
    M-->>P: ipcRenderer.return
    P-->>R: Promise resolved

    Note over R,M: ─── Main → Renderer (Event 推送) ───

    loop 压缩进行中
        M-->>P: ipcRenderer.send<br/>compress:progress
        P-->>R: onCompressProgress()
        R->>R: 更新进度条
    end

    M-->>P: ipcRenderer.send<br/>compress:result
    P-->>R: onCompressResult()
    R->>R: 更新 Store 状态

    Note over R,M: ─── 文件操作 ───

    R->>P: file:open-dialog / file:save-dialog
    P->>M: ipcRenderer.invoke
    M->>M: dialog.showOpenDialog<br/>/ dialog.showSaveDialog
    M-->>R: 返回路径

    Note over R,M: ─── 历史记录 (App 启动) ───

    R->>P: history:load
    P->>M: ipcRenderer.invoke
    M->>M: 读取 history.json
    M-->>R: HistoryRecord[]

    Note over R,M: ─── 历史记录 (压缩完成后) ───

    R->>P: history:add
    P->>M: ipcRenderer.invoke
    M->>M: 追加到 history.json
    M-->>R: { success }
```

### IPC 通道一览表

| 通道名称 | 方向 | 类型 | 参数 / 返回值 |
|---|---|---|---|
| `file:open-dialog` | R → M | invoke | args: `filters[]` → returns: `string[]` (路径) |
| `file:save-dialog` | R → M | invoke | args: `{ defaultPath }` → returns: `string` |
| `file:read-image-info` | R → M | invoke | args: `{ filePath }` → returns: `{ width, height, size, format }` |
| `file:save-to-path` | R → M | invoke | args: `{ compressedPath, targetPath }` → returns: `{ success, finalPath }` |
| `compress:start` | R → M | invoke | args: `{ id, filePath, options }` → returns: `{ id, status }` |
| `compress:batch` | R → M | invoke | args: `{ images[], options }` → returns: `{ batchId, status }` |
| `compress:cancel` | R → M | invoke | returns: `{ success }` |
| `compress:progress` | M → R | Event | payload: `{ id, progress }` |
| `compress:result` | M → R | Event | payload: `{ id, compressedPath, compressedSize, savedPercent }` |
| `history:load` | R → M | invoke | returns: `HistoryRecord[]` |
| `history:add` | R → M | invoke | args: `HistoryRecord` → returns: `{ success }` |
| `history:delete` | R → M | invoke | args: `{ id }` → returns: `{ success }` |
| `history:clear` | R → M | invoke | returns: `{ success }` |
| `history:export` | R → M | invoke | returns: `{ success, exportedPath }` |
| `history:import` | R → M | invoke | returns: `{ success, count }` |

## 五、状态流转图

```mermaid
stateDiagram-v2
    [*] --> pending: 上传图片

    state pending {
        [*] --> pending_format_ok
        [*] --> pending_too_large: 文件过大
        [*] --> pending_bad_format: 格式不支持

        pending_too_large --> [*]: 删除
        pending_bad_format --> [*]: 删除
    }

    pending --> compressing: 点击开始压缩
    compressing --> done: 压缩完成
    compressing --> error_compress: 压缩失败

    done --> done_in_history: 写入历史记录
    done_in_history --> [*]: 删除 / 导出保存

    error_compress --> [*]: 删除

    state error_compress {
        [*] --> error_unsupported: 格式不支持
        [*] --> error_corrupt: 文件损坏
        [*] --> error_save_fail: 保存失败
    }

    compressing --> compressing: 取消压缩
    compressing --> [*]: 取消后删除
```
